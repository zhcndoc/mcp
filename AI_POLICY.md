# AI 贡献政策

> [!IMPORTANT]
>
> 如果你在为 Model Context Protocol 做贡献时使用了**任何形式的 AI 辅助**，
> 必须在 pull request 或 issue 中披露这一点。

我们欢迎并鼓励使用 AI 工具来帮助改进 Model Context Protocol。许多有价值的贡献都在 AI 辅助下得到了增强，例如代码生成、问题检测和
功能定义。

尽管如此，如果你在为 Model Context Protocol 做贡献时使用了任何形式的 AI 辅助（例如 Claude Code、ChatGPT 之类的代理），
**必须在 pull request 或 issue 中披露这一点**，并说明 AI 辅助的使用程度（例如文档
注释还是代码生成）。

如果你的 PR 回复或评论是由 AI 生成的，也请一并披露。

作为例外，简单的空格调整或拼写修复无需披露，只要这些更改
仅限于代码的小部分或简短短语。

披露示例如下：

> This PR was written primarily by Claude Code.

或者更详细的披露：

> 我咨询了 ChatGPT 以理解代码库，但解决方案
> 完全由我本人手动编写。

不披露这一点，首先是对拉取请求另一端的人类操作员不礼貌，
而且也会使人难以判断应该对该贡献施加多大审查力度。

在一个理想的世界里，AI 辅助会产出与任何人类相当或更高质量的工作。但
我们如今并不生活在这样的世界中，而且在大多数没有人类监督或专业知识参与的情况下，它生成的代码或更改往往无法被合理地维护或演进。

## 我们在寻找什么

在提交 AI 辅助的贡献时，请确保它们包含：

- **清晰披露 AI 的使用** - 你明确说明了 AI 的使用，以及你在多大程度上
  将其用于此次贡献
- **人类理解** - 你本人理解这些更改的作用
- **清晰的理由** - 你可以解释为什么需要这项更改，以及它如何符合 Model
  Context Protocol 的目标
- **具体证据** - 包括测试用例、场景或示例，以展示
  改进效果
- **你自己的分析** - 分享你对端到端体验的看法

## 我们会关闭什么

我们保留关闭那些看起来未遵守披露政策的提交的权利。

## 适用范围

本政策适用于
[modelcontextprotocol](https://github.com/modelcontextprotocol) 组织中的所有仓库，包括
规范、SDK、参考服务器和工具。
