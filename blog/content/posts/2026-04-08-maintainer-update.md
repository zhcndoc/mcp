我很高兴分享维护者团队的两项更新：**Clare Liguori** 将加入 [Core Maintainer](https://modelcontextprotocol.io/community/governance#current-core-maintainers) 组，**Den Delimarsky** 将与我一起担任 [Lead Maintainer](https://modelcontextprotocol.io/community/governance#current-lead-maintainers)。

当我们在去年夏天[引入 MCP 治理模型](/posts/2025-07-31-governance-for-mcp/)时，目标是确保协议能够持续发展，而不会因某一个人成为瓶颈。这一点在经历了两个规范版本发布、迁移到 [Agentic AI Foundation (AAIF)](/posts/2025-12-09-mcp-joins-agentic-ai-foundation/)，以及 SEP 数量稳步增长之后，依然表现良好，而这些变化也为项目提供了迎接下一阶段所需的领导力。

## 欢迎 Clare Liguori 加入 Core Maintainer

<img src="/posts/images/maintainer-update-2026-04/clare.webp" alt="Clare Liguori" width="400" />

我很高兴欢迎 [Clare Liguori](https://github.com/clareliguori) 加入 [Core Maintainer](https://modelcontextprotocol.io/community/governance#current-core-maintainers) 团队。

Clare 是 Amazon Web Services 的高级首席工程师，她主要从事 agentic AI 开发者工具方面的工作。她目前专注于 [Kiro](https://kiro.dev/) 和 Strands Agents SDK，在 AWS 超过十年的工作中，她还参与过 AWS Proton、Amazon ECS、AWS Code Suite 以及多个开源项目。

Clare 一直在将自己在开发者工具和 agent 运行时方面的深厚经验直接带入 MCP 的设计工作，尤其是在[非请求任务](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/2229)、agent 执行模型，以及 [Triggers & Events 工作组](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/2459)相关讨论中。她亲身了解该协议在为大量开发者提供服务的生产级 agent 运行时内部应当是什么样子。随着这些提案在未来一年内继续推进 SEP 流程，这种视角将帮助规范始终立足于客户端实现者真正需要的内容。很高兴她能加入讨论。

欢迎你，Clare！

## Den Delimarsky 加入担任首席维护者

<img src="/posts/images/maintainer-update-2026-04/den.jpeg" alt="Den Delimarsky" width="400" />

[Den Delimarsky](https://github.com/localden) 正从 Core Maintainer 晋升为 Lead Maintainer，与我共同承担这一角色。

Den 是 Anthropic 的技术员工成员，他负责整个 MCP 生态系统中的工作：规范、SDK、治理，以及围绕这一切的开发者体验。在加入 Anthropic 之前，他曾担任微软 CoreAI 部门的首席产品工程师，并在开发者工具、SDK 和安全工具方面积累了丰富经验。

Den 的工作最显著体现在授权和安全领域。他共同撰写了[授权规范](https://modelcontextprotocol.io/specification/draft/basic/authorization)，将 [RFC 8707 Resource Indicators](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/734) 引入规范，并通过 [SEP-835](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/835)、[SEP-1024](https://modelcontextprotocol.io/seps/1024-mcp-client-security-requirements-for-local-server-) 以及 [SEP-2350](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/2350) 系列提案等持续推进这一方向。他还主导了 [2025-11-25 规范发布](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1887)，共同领导安全兴趣小组，并构建了一个[贡献追踪器](https://localden.github.io/mcp-repo-data-tracker/)，让 SDK 和规范维护者能够鸟瞰项目各个仓库中的活动情况。

我很高兴 Den 能以这一角色成为合作伙伴，也期待我们未来几年一起为协议做出更多成果。

## 接下来

MCP 仍在以我们任何人都无法预料的速度增长，而这种增长离不开那些每天推动它前进的人：SDK 维护者、工作组协调人，以及每一位撰写或审阅 SEP 的人。谢谢你们。

在这支团队就位之后，我很期待继续与维护者团队并肩合作，推动协议演进，并确保 MCP 能跟上社区对它的需求。如果你想参与其中，[治理文档](https://modelcontextprotocol.io/community/governance)会说明项目是如何组织的，[Contributor Ladder](https://modelcontextprotocol.io/community/contributor-ladder) 展示了人们如何成长并承担这些角色，[路线图](https://modelcontextprotocol.io/development/roadmap) 展示了我们的前进方向，而我们的 [Discord](https://modelcontextprotocol.io/community/communication#discord) 是与参与该项目的人交流的最佳场所。来和我们一起构建吧。
