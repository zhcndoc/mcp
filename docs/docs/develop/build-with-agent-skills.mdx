[Agent 技能](https://agentskills.io/home) 是可移植的指令集，
为 AI 编码助手提供特定任务的领域知识。对于 MCP 开发，
它们编码了设计决策（部署模型、工具模式、认证），以便
你的 Agent 可以询问你的用例并搭建一个合适的服务器骨架。

## 可用技能

一组参考的 MCP 开发技能可作为
[`mcp-server-dev` 插件](https://github.com/anthropics/claude-plugins-official/tree/main/plugins/mcp-server-dev) 获取。
它提供了三个组合技能：

| 技能               | 用途                                                                                                                 |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| `build-mcp-server` | 入口点。询问用例，选择部署模型和工具设计模式，路由到专用技能。 |
| `build-mcp-app`    | 添加交互式 UI 小部件（表单、选择器、仪表板），在聊天中内联渲染。                                       |
| `build-mcpb`       | 将本地 stdio 服务器与其运行时打包，以便用户无需 Node 或 Python 即可安装。                          |

每个技能都附带一个 `SKILL.md` 文件以及一个 `references/` 文件夹，其中包含支持
材料（认证流程、工具设计模式、小部件模板、清单 schema），
Agent 可按需读取这些文件。这些文件遵循开放格式，适用于
任何实现该标准的 Agent。例如，要在 Claude
Code 中安装它们：

```bash
/plugin marketplace add anthropics/claude-plugins-official
/plugin install mcp-server-dev
```

对于其他 Agent，请检查你的技能或扩展目录，或克隆
[技能目录](https://github.com/anthropics/claude-plugins-official/tree/main/plugins/mcp-server-dev/skills)
（`SKILL.md` 加 `references/`）到你的 Agent 的技能位置。

## 开始构建

安装技能后，请你的 Agent 帮助你构建一个 MCP 服务器。该
入口技能由自然语言请求触发，或者你可以使用
Agent 的技能调用语法直接调用它。

该技能在编写任何代码之前会运行一个简短的发现阶段。预计
会问到以下内容：

- **连接对象** — 云 API、本地进程、文件系统、硬件
- **使用者** — 仅你自己、你的团队，或任何安装它的人
- **功能范围** — 少数操作还是封装大型 API
- **用户交互需求** — 纯文本结果、通过
  [需求诱导](/specification/draft/client/elicitation) 的结构化输入，或丰富的 UI 小部件
- **上游认证** — API 密钥、OAuth 2.0，或无

如果你的开场消息已经涵盖了这些，Agent 会跳过并直接给出建议。

## 部署路径

基于发现结果，该技能会推荐四种路径之一并相应地搭建骨架：

**远程 [Streamable HTTP](/specification/draft/basic/transports/streamable-http)**
是任何封装云 API 的默认选择。零安装摩擦，一个部署服务所有用户，
并且 OAuth 流程可以正常工作，因为服务器
可以处理重定向和令牌存储。参考技能包含
Cloudflare Workers 和可移植 Express/FastMCP 设置的脚手架。

**[MCP 应用](/extensions/apps/overview)** 通过交互式
小部件扩展服务器，这些小部件在聊天中渲染，例如可搜索选择器、图表和实时
仪表板。当
[需求诱导](/specification/draft/client/elicitation) 的扁平表单限制
不适用时，该技能会移交至 `build-mcp-app`。

**[MCP 捆绑包 (MCPB)](https://github.com/modelcontextprotocol/mcpb)** 将
本地服务器与其运行时打包为单个 `.mcpb` 归档文件，以便用户
无需设置 Node 或 Python 即可安装。当服务器必须
访问用户机器时使用此路径：读取本地文件、驱动桌面应用，或
与 localhost 服务通信。该技能会移交至 `build-mcpb`。

**Local [stdio](/specification/draft/basic/transports/stdio)** 仍可用于
原型开发，并注明在准备
分发时可升级到 MCPB。
