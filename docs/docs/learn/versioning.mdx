模型上下文协议使用基于字符串的版本标识符，格式为 `YYYY-MM-DD`，用于表示最后一次发生向后不兼容更改的日期。

<Info>

只要更改保持向后兼容性，协议更新时协议版本将 _ 不 _ 会递增。这允许在保持互操作性的同时进行增量改进。

</Info>

## 修订版本

修订版本可能标记为：

- **草案**：进行中的规范，尚未准备好供使用。
- **当前**：当前的协议版本，已准备好使用，并且可能会继续接收向后兼容的更改。
- **最终**：过去的、完整的规范，将不再更改。

**当前** 协议版本是 [**2025-11-25**](/specification/2025-11-25/)。

## 特性状态

规范的单个特性还可能被标记为
**已弃用**，根据
[特性生命周期和弃用策略](/community/feature-lifecycle)：
该特性仍然是规范的一部分，但计划移除。
已弃用特性会记录迁移路径（或说明无需迁移），并且在规范中至少保留十二个月，或者根据该策略的
[加速移除例外](/community/feature-lifecycle#expedited-removal) 至少保留九十天，
之后才有资格被移除，随后它们可能会在未来的修订版中被 **移除**。

当前已弃用的特性列在
[已弃用特性注册表](/specification/draft/deprecated) 中。

## 协商

版本协商发生在 [初始化](/specification/latest/basic/lifecycle#initialization) 期间。客户端和服务器 **可以** 同时支持多个协议版本，但它们 **必须** 就会话使用的单个版本达成一致。

如果版本协商失败，协议会提供适当的错误处理，允许客户端在找不到与服务器兼容的版本时优雅地终止连接。
