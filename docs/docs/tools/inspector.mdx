[MCP Inspector](https://github.com/modelcontextprotocol/inspector) 是一个交互式开发者工具，用于测试和调试 MCP 服务器。虽然 [调试指南](/docs/tools/debugging) 将 Inspector 作为整体调试工具包的一部分进行了介绍，但本文档提供了对 Inspector 功能和能力的详细探索。

## 入门指南

### 安装和基本用法

Inspector 可以直接通过 `npx` 运行，无需安装：

```bash
npx @modelcontextprotocol/inspector <command>
```

```bash
npx @modelcontextprotocol/inspector <command> <arg1> <arg2>
```

#### 检查来自 npm 或 PyPI 的服务器

从 [npm](https://npmjs.com) 或 [PyPI](https://pypi.org) 启动服务器包的常用方法。

<Tabs>

  <Tab title="npm 包">

```bash
npx -y @modelcontextprotocol/inspector npx <package-name> <args>
# 例如
npx -y @modelcontextprotocol/inspector npx @modelcontextprotocol/server-filesystem /Users/username/Desktop
```

  </Tab>

  <Tab title="PyPI 包">

```bash
npx @modelcontextprotocol/inspector uvx <package-name> <args>
# 例如
npx @modelcontextprotocol/inspector uvx mcp-server-git --repository ~/code/mcp/servers.git
```

  </Tab>
</Tabs>

#### 检查本地开发的服务器

要检查本地开发或作为仓库下载的服务器，最常见的方法是：

<Tabs>
  <Tab title="TypeScript">

```bash
npx @modelcontextprotocol/inspector node path/to/server/index.js args...
```

  </Tab>
  <Tab title="Python">

```bash
npx @modelcontextprotocol/inspector \
  uv \
  --directory path/to/server \
  run \
  package-name \
  args...
```

  </Tab>
</Tabs>

请仔细阅读附带的 README 以获取最准确的说明。

## 功能概述

<Frame caption="MCP Inspector 界面">
  <img src="/images/mcp-inspector.png" />
</Frame>

Inspector 提供了多种功能用于与您的 MCP 服务器交互：

### 服务器连接面板

- 允许选择用于连接服务器的 [传输方式](/specification/latest/basic/transports)
- 对于本地服务器，支持自定义命令行参数和环境

### 资源标签页

- 列出所有可用资源
- 显示资源元数据（MIME 类型、描述）
- 允许检查资源内容
- 支持订阅测试

### 提示词标签页

- 显示可用的提示词模板
- 显示提示词参数和描述
- 支持使用自定义参数测试提示词
- 预览生成的消息

### 工具标签页

- 列出可用工具
- 显示工具模式和描述
- 支持使用自定义输入测试工具
- 显示工具执行结果

### 通知面板

- 展示从服务器记录的所有日志
- 显示从服务器接收到的通知

## 最佳实践

### 开发工作流

1. 开始开发
   - 使用您的服务器启动 Inspector
   - 验证基本连接
   - 检查能力协商

2. 迭代测试
   - 进行服务器更改
   - 重建服务器
   - 重新连接 Inspector
   - 测试受影响的特征
   - 监控消息

3. 测试边界情况
   - 无效输入
   - 缺失提示词参数
   - 并发操作
   - 验证错误处理和错误响应
