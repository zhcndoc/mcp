title: 什么是模型上下文协议 (MCP)?
sidebarTitle: 什么是 MCP?
---

MCP（模型上下文协议）是一个用于将 AI 应用程序连接到外部系统的开源标准。

使用 MCP，像 Claude 或 ChatGPT 这样的 AI 应用程序可以连接到数据源（例如本地文件、数据库）、工具（例如搜索引擎、计算器）和工作流（例如专用提示）——使它们能够访问关键信息并执行任务。

可以将 MCP 想象成 AI 应用程序的 USB-C 端口。正如 USB-C 提供了一种连接电子设备的标准化方式一样，MCP 提供了一种将 AI 应用程序连接到外部系统的标准化方式。

## MCP 能实现什么？

- Agent 可以访问您的 Google Calendar 和 Notion，充当更个性化的 AI 助手。
- Claude Code 可以使用 Figma 设计生成整个 Web 应用程序。
- 企业聊天机器人可以连接到组织内的多个数据库，使用户能够通过聊天分析数据。
- AI 模型可以在 Blender 上创建 3D 设计，并使用 3D 打印机打印出来。

## 为什么 MCP 很重要？

根据您在生态系统中的位置，MCP 可以带来一系列好处。

- **开发者**：MCP 减少了构建或集成 AI 应用程序或 Agent 时的开发时间和复杂性。
- **AI 应用程序或 Agent**：MCP 提供了访问数据源、工具和应用程序生态系统的权限，这将增强功能并改善最终用户体验。
- **最终用户**：MCP 带来了功能更强大的 AI 应用程序或 Agent，它们可以访问您的数据并在必要时代表您采取行动。

## 广泛的生态系统支持

MCP 是一个开放协议，得到广泛的客户端和服务器支持。像 [Claude](https://claude.com/docs/connectors/building) 和 [ChatGPT](https://developers.openai.com/api/docs/mcp/) 这样的 AI 助手，像 [Visual Studio Code](https://code.visualstudio.com/docs/copilot/chat/mcp-servers)、[Cursor](https://cursor.com/docs/context/mcp)、[MCPJam](https://docs.mcpjam.com/getting-started) 这样的开发工具，以及 [许多其他工具](/clients) 都支持 MCP —— 使得一次构建即可随处集成变得容易。
