本文档列出了自上一个修订版 [2025-06-18](/specification/2025-06-18) 以来对模型上下文协议 (MCP) 规范所做的变更。

## 主要变更

1. 增强授权服务器发现功能，支持 [OpenID Connect Discovery 1.0](https://openid.net/specs/openid-connect-discovery-1_0.html)。(PR [#797](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/797))
2. 允许服务器将图标作为工具、资源、资源模板和提示的额外元数据暴露出来 ([SEP-973](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/973))。
3. 通过 `WWW-Authenticate` 增强授权流程，支持增量范围同意 ([SEP-835](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/835))
4. 提供关于工具名称的指导 ([SEP-986](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1603))
5. 更新 `ElicitResult` 和 `EnumSchema` 以使用更基于标准的方法，并支持带标题、无标题、单选和多选枚举 ([SEP-1330](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1330))。
6. 添加了对 [URL 模式诱导](/specification/2025-11-25/client/elicitation#url-elicitation-requests) 的支持 ([SEP-1036](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/887))
7. 通过 `tools` 和 `toolChoice` 参数向采样添加工具调用支持 ([SEP-1577](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1577))
8. 添加对 OAuth Client ID Metadata Documents 的支持，作为一种推荐的客户端注册机制 ([SEP-991](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/991), PR [#1296](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1296))
9. 添加对 [任务](/specification/2025-11-25/basic/utilities/tasks) 的实验性支持，以启用通过轮询和延迟结果检索跟踪持久请求 ([SEP-1686](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1686))。

## 次要变更

1. 澄清使用 stdio 传输的服务器可以使用 stderr 进行所有类型的日志记录，而不仅仅是错误消息 (PR [#670](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/670))。
2. 向 `Implementation` 接口添加可选的 `description` 字段，以与 MCP 注册表 server.json 格式保持一致，并在初始化期间提供人类可读的上下文。
3. 澄清服务器必须在 Streamable HTTP 传输中为无效的 Origin 头响应 HTTP 403 Forbidden。(PR [#1439](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1439))
4. 更新了 [安全最佳实践指导](https://modelcontextprotocol.io/specification/draft/basic/security_best_practices)。
5. 澄清输入验证错误应作为工具执行错误返回，而不是协议错误，以启用模型自我纠正 ([SEP-1303](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1303))。
6. 支持轮询 SSE 流，允许服务器随时断开连接 ([SEP-1699](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1699))。
7. 澄清 SEP-1699：GET 流支持轮询，无论流来源如何恢复始终通过 GET，事件 ID 应编码流身份，断开连接包括服务器发起的关闭 (问题 [#1847](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1847))。
8. 使 OAuth 2.0 受保护资源元数据发现与 RFC 9728 保持一致，使 `WWW-Authenticate` 头成为可选，并回退到 `.well-known` 端点 ([SEP-985](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/985))。
9. 添加对诱导模式所有原始类型（字符串、数字、枚举）默认值的支持 ([SEP-1034](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1034))。
10. 确立 JSON Schema 2020-12 为 MCP 模式定义的默认方言 ([SEP-1613](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1613))。

## 其他模式变更

1. 将请求负载与 RPC 方法定义解耦为独立的参数模式。([SEP-1319](https://github.com/modelcontextprotocol/specification/issues/1319), PR [#1284](https://github.com/modelcontextprotocol/specification/pull/1284))
