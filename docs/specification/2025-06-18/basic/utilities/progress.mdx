模型上下文协议 (MCP) 支持通过通知消息对长时间运行的操作进行可选的进度跟踪。任何一方都可以发送进度通知以提供有关操作状态的更新。

## 进度流程

当一方想要_接收_请求的进度更新时，它会在请求元数据中包含一个 `progressToken`。

- 进度令牌 **MUST** 为字符串或整数值
- 进度令牌可以由发送方通过任何方式选择，但在所有活动请求中 **MUST** 是唯一的。

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "some_method",
  "params": {
    "_meta": {
      "progressToken": "abc123"
    }
  }
}
```

接收方 **MAY** 随后发送包含以下内容的进度通知：

- 原始进度令牌
- 当前的进度值
- 可选的 "total" 值
- 可选的 "message" 值

```json
{
  "jsonrpc": "2.0",
  "method": "notifications/progress",
  "params": {
    "progressToken": "abc123",
    "progress": 50,
    "total": 100,
    "message": "Reticulating splines..."
  }
}
```

- `progress` 值 **MUST** 随每个通知增加，即使总数未知。
- `progress` 和 `total` 值 **MAY** 为浮点数。
- `message` 字段 **SHOULD** 提供相关的可读进度信息。

## 行为要求

1. 进度通知 **MUST** 仅引用以下令牌：
   - 在活动请求中提供的
   - 与进行中的操作关联的

2. 进度请求的接收方 **MAY**：
   - 选择不发送任何进度通知
   - 以其认为合适的任何频率发送通知
   - 如果未知则省略 total 值

```mermaid
sequenceDiagram
    participant Sender
    participant Receiver

    Note over Sender,Receiver: Request with progress token
    Sender->>Receiver: Method request with progressToken

    Note over Sender,Receiver: Progress updates
    Receiver-->>Sender: Progress notification (0.2/1.0)
    Receiver-->>Sender: Progress notification (0.6/1.0)
    Receiver-->>Sender: Progress notification (1.0/1.0)

    Note over Sender,Receiver: Operation complete
    Receiver->>Sender: Method response
```

## 实现说明

- 发送方和接收方 **SHOULD** 跟踪活动进度令牌
- 双方 **SHOULD** 实施速率限制以防止泛滥
- 进度通知 **MUST** 在完成后停止
