此矩阵展示了哪些 MCP 客户端支持每个 [官方扩展](/extensions/overview)。扩展始终需要主动启用——只有当客户端和服务器在 [初始化握手](/extensions/overview#negotiation) 期间都声明支持时，客户端才会使用扩展。

<Note>

此列表由社区维护。如果您发现任何不准确之处或希望添加或更新信息，请 [提交拉取请求](https://github.com/modelcontextprotocol/modelcontextprotocol/pulls)。

</Note>

## 扩展概述

| 扩展                                                                                   | 标识符                                                   | 描述                                                     |
| -------------------------------------------------------------------------------------- | -------------------------------------------------------- | -------------------------------------------------------- |
| [MCP Apps](/extensions/apps/overview)                                                  | `io.modelcontextprotocol/ui`                             | 在对话中内联渲染的交互式 HTML 界面                       |
| [OAuth 客户端凭证](/extensions/auth/oauth-client-credentials)                          | `io.modelcontextprotocol/oauth-client-credentials`       | 无需交互式用户登录的机器到机器认证                       |

<Note>

Auth 扩展支持（OAuth Client Credentials 和 Enterprise-Managed Authorization）是单独跟踪的，不属于核心 MCP 授权功能（DCR、CIMD）。请查看各扩展的规范以及 [ext-auth 仓库](https://github.com/modelcontextprotocol/ext-auth) 以获取最新的实现状态。

</Note>

## 为您的客户端添加扩展支持

如果您正在构建 MCP 客户端并希望实现扩展支持：

1. 查阅扩展规范（例如在 [ext-auth](https://github.com/modelcontextprotocol/ext-auth) 或 [ext-apps](https://github.com/modelcontextprotocol/ext-apps) 仓库中）
2. 在 `initialize` 能力的 `extensions` 字段中声明支持
3. 实现扩展的协议要求
4. 提交拉取请求以更新此矩阵

请参阅 [扩展概述](/extensions/overview#negotiation) 以了解能力协商机制的详细信息。
