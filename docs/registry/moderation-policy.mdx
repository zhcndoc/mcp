**TL;DR**：MCP 注册表非常宽松！我们只移除非法内容、恶意软件、垃圾信息和完全无法使用的服务器。

## 范围

本政策适用于位于 `registry.modelcontextprotocol.io` 的官方 MCP 注册表。

子注册表可能有自己的审核政策。如果您对特定子注册表上的内容有疑问，请直接联系他们。

## 免责声明

MCP 注册表**不**对审核做出保证，使用者应假设几乎没有或完全没有审核。

MCP 注册表是一个社区支持的项目，我们的主动审核能力有限。我们主要依赖上游包注册表（如 NPM、PyPI 和 Docker）或下游子注册表（如 GitHub MCP 注册表）来进行更深入的审核。

这意味着 MCP 注册表中可能存在根据本政策应被移除但尚未被移除的内容。使用者应相应地对待抓取的数据。

## 我们移除什么

我们将移除包含以下内容的服务器：

- 非法内容，包括淫秽内容、侵犯版权和黑客工具
- 恶意软件，无论意图如何
- 垃圾信息，尤其是破坏注册表的大批量创建的服务器。例如：
  - 同一服务器以不同名称多次提交
  - 除了提供带有某些营销文案的固定响应外什么都不做的服务器
  - 描述中塞满了营销文案且实现不相关的服务器
- 无法运行的服务器

## 我们不移除什么

通常，我们相信保持注册表开放并将审核推给子注册表。因此我们**不会**移除：

- 低质量或有错误的服务器
- 存在安全漏洞的服务器
- 与其他服务器做同样事情的服务器
- 提供或包含成人内容的服务器

## 移除如何运作

当我们移除服务器时，我们将服务器的 `status` 设置为 `"deleted"`，但服务器的元数据仍可通过 MCP 注册表 API 访问。聚合器随后可以从其索引中移除该服务器。

在极端情况下，我们可能会覆盖或擦除服务器的元数据。例如，如果元数据本身是非法的。

## 申诉

认为我们犯了错误？在我们的 [GitHub 仓库](https://github.com/modelcontextprotocol/registry) 上开启一个 issue，包含：

- 服务器名称
- 您认为该服务器不符合上述移除标准的原因

## 本政策的变更

我们仍在学习如何最好地运行 MCP 注册表！因此，我们未来可能会更改此政策。
