## 组别类型

**工作组**

## 使命声明

注册工作组旨在构建并维护官方 MCP 注册表——一个面向公开可用 MCP 服务器的开放目录和 API——以便客户端、子注册表和终端用户能够自信地发现、评估并安装服务器。该工作组负责注册表服务、`server.json` 模式、注册表 API 规范，以及进一步分发服务器元数据的子注册表生态系统。

## 范围

### 范围内

- **注册表服务**：托管于 `registry.modelcontextprotocol.io` 的注册表的运行、可靠性与演进，包括正常运行时间、监控和事件响应。
- **注册表 API 规范**：定义任何注册表（官方或私有）如何公开服务器元数据的 OpenAPI 规范。
- **`server.json` 模式**：用于描述 MCP 服务器标识、包、运行时配置和能力的标准化格式——与 Server Card 工作组协调，以保持 Server Card 作为一个一致的子集。
- **客户端 SDK**：生成式或手工维护的客户端库，便于客户端和子注册表集成注册表 API。
- **发布与信任**：认证流程（GitHub OAuth、GitHub OIDC、DNS/HTTP 验证）、命名空间所有权、审核工具以及社区驱动的标记。
- **采用与推广**：文档、入门指南和推广，以提升目录覆盖率。
- **问题分流与自动化**：注册表仓库的标签系统、分流和贡献者工作流。

### 范围外

- 任何与运行时相关的 MCP 协议规范方面（由核心维护者和其他工作组负责）。
- Server Card 格式和发现机制（由 Server Card 工作组负责；本工作组协调 `server.json` 对齐）。
- 代表 MCP 客户端或终端用户对不同 MCP 服务器实现进行排序/选择。
- 托管、分发或执行 MCP 服务器代码或二进制文件——注册表是元数据目录，而不是包注册表。
- 对交付可供企业使用或可复用的注册表实现的任何承诺。该代码库仅支持此实例，不打算用于外部部署。

### 相关组别

- **Server Card 工作组** —— `server.json` 和 Server Card 必须保持一致；注册表将为已发布条目公开 Server Card + 本地包相关元数据。需要紧密协调以避免模式分歧。

### 成功标准

- 通过自动化监控和告警实现注册表正常运行时间 ≥ 99.9%。
- `server.json` 模式与 Server Card 格式对齐，且没有非预期的分歧。
- 大多数流行的、公开可用的 MCP 服务器已发布到注册表。
- 至少提供一个客户端 SDK（生成式或维护式）供注册表消费者使用。
- 注册表 API v1 规范已最终确定并保持稳定。
- 活跃的子注册表生态系统正在使用官方注册表 API。
