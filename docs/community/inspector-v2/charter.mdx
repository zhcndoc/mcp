## 使命声明

Inspector V2 工作组正在构建 Inspector V2，这是一款全新的基于 Web 的 MCP 检查器，从零开始重新设计，以提升可维护性和可靠性。该工作组提供共享的 Inspector Core 架构，最大化 Web、CLI 和 TUI 实现之间的代码复用，并配套一套完整的测试机制。由于这项工作涵盖 UI、协议工具和测试基础设施，而这些今天并不由任何单一维护者负责，因此需要跨维护者协作。

## 范围

### 范围内

- `modelcontextprotocol/inspector` 仓库，包括：
- **Inspector Core** —— 一种新的共享代码架构，为所有 Inspector 前端提供通用的 MCP 和协议接口。
- **Web Inspector UI** —— 基于 Mantine 和 TypeScript 构建的浏览器端检查器。
- **CLI Inspector** —— 共享 Inspector Core 的命令行界面。
- **TUI Inspector** —— 共享 Inspector Core 的终端界面。
- **Testing apparatus** —— 跨所有 Inspector 界面的共享测试工具、测试夹具和集成测试。
- 将 `main` 分支上的现有 Inspector 弃用，并迁移到 `v1.x` 维护分支。
- 一旦可用，在 Inspector Core 中采用 MCP TypeScript SDK V2。

### 范围外

- 核心 MCP 规范。
- MCP SDK 维护（TypeScript、Python 或任何其他语言的 SDK）。
- MCP 服务器实现。

### 相关工作组

- **SDK WG** —— Inspector Core 会使用 TypeScript SDK；采用 SDK V2 需要协调。
- **MCP Apps WG** —— 围绕客户端/应用易用性和检查工作流的共享表面区域。
- **Auth WG** —— Inspector 在连接受保护服务器时所使用的认证流程。
- **Registry WG** —— Inspector 向用户展示的发现和元数据界面。
