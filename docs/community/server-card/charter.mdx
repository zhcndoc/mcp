## 小组类型

**工作组**

## 使命陈述

服务器卡片工作组旨在定义机制，以促进通过通用发现机制发现和使用的 MCP 服务器，并就其如何融入更广泛的围绕发现的 AI 标准化工作提供指导。具体来说，工作组将定义什么是 MCP 服务器卡片，服务器卡片必须遵循的标准化文档格式，以及客户端如何发现给定服务器的服务器卡片。

## 范围

### 范围内

- **规范工作**：定义什么是 MCP 服务器卡片以及服务器卡片文档具体格式的 SEP 或扩展。
- **发现机制**：指定如何发现 MCP 服务器卡片文档（well-known URL、基于资源的发现等）。
- **跨生态系统协调**：向 [AI 卡片](https://github.com/Agent-Card/ai-card) 工作提出关于如何与 MCP 服务器卡片交互的建议。
- **文档**：涵盖服务器卡片编写和使用的规范部分和指南。

### 范围外

- 更改 MCP 初始化握手或传输层。
- 通用 MCP 服务器注册表或目录（由注册表工作组拥有）。
- 在定义 MCP 范围的 i18n 方法之前，服务器卡片内容的国际化。

### 相关小组

- **注册表工作组** — 服务器卡片格式应尽可能保持与 `server.json` 子集接近；需要协调以避免分歧。
- **AI 卡片工作**（外部）— 正在讨论 AI 目录是否可以以及如何直接链接到 MCP 服务器卡片。
