每个 MCP 工作组和兴趣组都必须维护一份遵循此结构的章程文档。章程存储在 [modelcontextprotocol 仓库](https://github.com/modelcontextprotocol/modelcontextprotocol) 中的 `docs/community/working-groups/<group-name>.mdx` 或 `docs/community/interest-groups/<group-name>.mdx`，并添加到 `docs/docs.json` 中。

章程用于记录特定于您所在组的信息。治理规则——领导要求、决策流程、会议要求、升级路径——在 [工作组和兴趣组](/community/working-interest-groups) 文档中定义并自动适用。不要在此处重复它们。

标记为 **(仅工作组)** 的部分是工作组必需的，对于兴趣组是可选的。

<Note>
  将下面的 markdown 复制到 `docs/community/working-groups/<group-name>.mdx`（如果是 IG，则复制到 `interest-groups/`）中，并替换占位文本。
</Note>

---

## 使命声明示例

**工作组：**

> 传输工作组旨在演进 MCP 的传输机制，以支持多样化的部署场景——从本地子进程通信到水平扩展的云部署——同时保持协议一致性和向后兼容性。

**兴趣组：**

> 企业兴趣组探索在企业环境中部署 MCP 的挑战，收集用例和需求以为未来的规范工作提供信息。
